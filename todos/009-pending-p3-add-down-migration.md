## Problem Statement

The database migration for adding frequency_stats columns lacks a corresponding rollback/down migration.

## Findings

**From:** data-migration-expert

**Migration file:** `backend/app/migrations/add_frequency_stats_to_appointment_prep_contexts.sql`

**Current migration only adds columns:**

```sql
ALTER TABLE appointment_prep_contexts
ADD COLUMN IF NOT EXISTS frequency_stats JSONB DEFAULT NULL;

ALTER TABLE appointment_prep_contexts
ADD COLUMN IF NOT EXISTS cooccurrence_stats JSONB DEFAULT NULL;
```

**Risk:** If deployment fails after migration runs, manual intervention is required.

## Proposed Solutions

### Option A: Add Down Migration (Recommended)

**Effort:** Small

Create a corresponding down migration file:

```sql
-- Rollback: remove stats columns from appointment_prep_contexts
ALTER TABLE appointment_prep_contexts
DROP COLUMN IF EXISTS frequency_stats;

ALTER TABLE appointment_prep_contexts
DROP COLUMN IF EXISTS cooccurrence_stats;
```

### Option B: Document Manual Rollback

**Risk:** Higher operational burden during incidents.

## Recommended Action

Add a down migration file.

## Technical Details

**New file:** `backend/app/migrations/add_frequency_stats_to_appointment_prep_contexts_down.sql`

**Note:** Stats can be regenerated by re-running Step 2, so data loss on rollback is acceptable.
